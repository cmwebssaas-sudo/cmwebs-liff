# V2 圖形化報表規格

## 目標

房東不需要讀 Google Sheets，即可掌握：

- 應收與實收
- 欠款
- 入住率
- 物件績效
- 租約到期
- 用電
- 報修
- 押金負債

## 建議檔案

```text
apps-script/V2_REPORTING_DASHBOARD.gs
public/landlord-reports.html
```

建議 API：

```text
landlord_reports_init
landlord_reports_export
```

## 篩選器

- Workspace
- 日期區間
- 本月／近 3 月／近 12 月／自訂
- 物件
- 房間
- 報表類型

## 第一版 KPI

- 本月應收
- 本月實收
- 未收金額
- 收款率
- 入住率
- 空房數
- 逾期帳單數
- 即將到期租約
- 待處理報修
- 待退押金

## 第一版圖表

1. 月份應收／實收／欠款折線圖
2. 欠款帳齡長條圖
   - 1–7 天
   - 8–15 天
   - 16–30 天
   - 31 天以上
3. 各物件收入與收款率
4. 入住率／空房率
5. 租約到期月份分布
6. 每月用電與電費
7. 報修狀態分布
8. 押金應退、已扣與已退

## 資料定義

- 應收：指定期間 `bill_status=issued` 的帳單總額
- 實收：核准付款或手動結清的金額
- 未收：應收－實收，不得只依帳單筆數
- 收款率：實收／應收；應收為 0 時顯示 `—`
- 入住率：有效租約房間數／可出租房間數
- 空房：非封存且沒有有效租約
- 欠款帳齡：以 Workspace 時區計算逾期天數
- 押金負債：尚未完成退租結算的可退押金

## UI 規則

- 圖表下方必須有數值表格 fallback
- 手機可橫向捲動，不壓縮到無法閱讀
- 顯示資料更新時間
- 無資料時顯示空狀態，不顯示 0 的誤導圖表
- 匯出 CSV 至少包含月報與物件績效

## 技術建議

- 可使用固定版本 Chart.js，但需 pin 版本
- 報表 API 一次回傳聚合資料，不讓前端自行掃描明細
- 建立月份聚合或 CacheService，避免每次全表掃描
- 第一版不加入 AI 分析

## 驗收

- 同一日期與物件篩選，KPI、圖表與 CSV 數字一致
- 與 Google Sheets 抽樣人工計算一致
- 12 個月資料在可接受時間內載入
- 不同 Workspace 絕不互相洩漏
